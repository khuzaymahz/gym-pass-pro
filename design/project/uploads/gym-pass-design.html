Screen specifications

14 screens · per-screen contracts

Each screen spec lists: route, state, components, behavior, empty/error states, and acceptance criteria. Build screens in the order given; earlier screens unblock later ones.

01
Login
/auth/login
AUTH · UNAUTH ONLY
Purpose
First screen a new or returning user sees. Collects phone number and requests OTP.
Key widgets
PhoneInput, PrimaryButton, editorial headline, animated "LIVE NOW" hero card with pulsing dot.
Validation
Phone must match ^0(77|78|79)\d{7}$. Normalize to +9627XXXXXXXXX before submit. Show error message in form-level error region (not a snackbar).
Behavior
On CTA tap: disable button, call authNotifier.requestOtp(phone). On success → push /auth/otp with phone as extra. On error → show inline message, re-enable button.
Accept if
Jordanian numbers accepted · invalid numbers blocked · works in RTL · haptic on submit.
02
OTP Verification
/auth/otp?phone={e164}
AUTH · 120s TIMER
Purpose
Verify phone ownership. Entry point for both new and returning users.
Components
OtpInput (6 cells), countdown timer (120s), resend row, dev hint ("Use 123456" in non-release builds only).
Behavior
Auto-submit on 6th digit. verifyOtp() returns isNewUser: true → push /auth/register; false → go('/'). Timer can restart via resend (disabled until 0).
Edge cases
Wrong OTP: shake digits animation + clear fields. Expired: block verify, show "Resend code". Lost network: retry with exponential backoff.
03
Register
/auth/register
AUTH · NEW USERS ONLY
Fields
Full name (req, ≥2 chars), email (opt, valid format), password (req, ≥8), confirm password (match), terms checkbox (req).
Behavior
On submit → authNotifier.completeRegistration(data) → on success, go('/').
Accept if
Validation firing inline · terms link opens in in-app browser · avatar initial derived from name's first letter.

Font strategy rule

When Directionality.of(context) == TextDirection.rtl, the theme swaps displayFontFamily to Cairo Black and body to Cairo Regular. Do NOT attempt to render Archivo glyphs for Arabic text — they don't exist.

Theme provider pattern

// core/theme/theme_provider.dart
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:shared_preferences/shared_preferences.dart';

final themeModeProvider = StateNotifierProvider<ThemeModeNotifier, ThemeMode>(
  (ref) => ThemeModeNotifier(),
);

class ThemeModeNotifier extends StateNotifier<ThemeMode> {
  static const _key = 'theme_mode';

  ThemeModeNotifier() : super(ThemeMode.dark) {
    _load();
  }

  Future<void> _load() async {
    final prefs = await SharedPreferences.getInstance();
    final raw = prefs.getString(_key) ?? 'dark';
    state = switch (raw) {
      'light' => ThemeMode.light,
      'system' => ThemeMode.system,
      _ => ThemeMode.dark,
    };
  }

  Future<void> setMode(ThemeMode mode) async {
    state = mode;
    final prefs = await SharedPreferences.getInstance();
    await prefs.setString(_key, mode.name);
  }
}
08

RTL handling

Flutter mirrors automatically — with caveats

Flutter auto-mirrors Padding, Row, Align, Positioned, etc. when Directionality is rtl. But icons, animations, numbers, and custom Transforms do NOT mirror. Follow these rules religiously.

● LTR (English)
G
Gold Pass · 45 visits
›
● RTL (Arabic)
ذ
اشتراك ذهبي · ٤٥ زيارة
›

RTL rules

  1. Use logical properties. Prefer EdgeInsetsDirectional over EdgeInsets, AlignmentDirectional.centerStart over Alignment.centerLeft. Use start/end — never left/right.
  2. Mirror chevrons & directional icons. chevron_right points the wrong way in RTL. Use Icons.arrow_forward_ios wrapped in a Transform that flips based on directionality, or use chevron_right in AR since it visually reads as "next" when mirrored. Default pattern:
    Transform(alignment: Alignment.center, transform: Matrix4.identity()..scale(isRtl ? -1.0 : 1.0, 1.0), child: Icon(Icons.chevron_right))
  3. Numbers stay LTR. Phone numbers, prices, OTP digits, timers — ALL stay in LTR visual order even in Arabic UI. Wrap them in Directionality(textDirection: TextDirection.ltr, child: ...).
  4. Format numbers with intl. Use NumberFormat('#,##0.00', 'ar_JO') to get Arabic-Indic numerals (٠١٢٣٤٥٦٧٨٩) when appropriate. For prices and phone numbers, use Latin numerals (more internationally readable).
  5. Logos are NEVER mirrored. Wrap the app logo in Directionality(textDirection: TextDirection.ltr, child: ...).
  6. Swipe gestures mirror. A "swipe to delete" from the right in LTR becomes a swipe from the left in RTL. Flutter's Dismissible handles this — but test it.
  7. Animations. Slide-in transitions should originate from the leading edge, not a hardcoded side. Use SlideTransition with Offset.fromDirection or check Directionality.of.
  8. QR scanner overlay. Scan-line animation moves top-to-bottom — this does not mirror. The "point at code" text label flips naturally.
  9. Currency symbol placement. EN: 45 JOD · AR: ٤٥ د.أ (currency after number in both). Use the priceFormat ARB string.
!
Common mistake

Setting textAlign: TextAlign.left breaks RTL. Always use TextAlign.start / TextAlign.end. The default for most Text widgets is already correct — only override when necessary.

RTL helper utility

// core/utils/rtl_helpers.dart
extension RtlContext on BuildContext {
  bool get isRtl => Directionality.of(this) == TextDirection.rtl;
}

class DirectionalIcon extends StatelessWidget {
  final IconData icon;
  final double? size;
  final Color? color;

  const DirectionalIcon(this.icon, {super.key, this.size, this.color});

  @override
  Widget build(BuildContext context) {
    return Transform(
      alignment: Alignment.center,
      transform: Matrix4.identity()..scale(context.isRtl ? -1.0 : 1.0, 1.0),
      child: Icon(icon, size: size, color: color),
    );
  }
}

Redirect logic

GoRouter(
  initialLocation: '/splash',
  redirect: (context, state) {
    final session = ref.read(authNotifierProvider);
    final loc = state.matchedLocation;

    final isAuthRoute = loc.startsWith('/auth');
    final isSplash    = loc == '/splash';

    if (session.isLoading && !isSplash) return '/splash';
    if (!session.isAuthenticated && !isAuthRoute && !isSplash) {
      return '/auth/login';
    }
    if (session.isAuthenticated && (isAuthRoute || isSplash)) {
      return '/';
    }
    return null;
  },
  routes: [ /* ...see routes.dart */ ],
);

01

Product overview

Context · what we're building

Gym Pass is a Jordan-based gym pass app. One subscription grants a member access to a network of gyms across a tiered system. Members browse gyms, subscribe to a tier, and check in at any included gym via QR code.

Core value proposition

  • One subscription unlocks many gyms — no separate memberships
  • Tiered access — Silver / Gold / Platinum / Diamond — each tier includes more gyms and more monthly visits
  • QR-based check-in — members scan a gym's QR code to enter, subject to subscription and tier validation
  • Jordan-first — Arabic-first UI, JOD currency, +962 phone format, local payment methods

Scope

◆ IN SCOPE (v1.0)

  • Member mobile app (iOS + Android)
  • Auth: phone OTP + registration
  • Browse gyms · filter by category
  • Subscribe to plan · payment stub
  • QR check-in flow
  • Profile & subscription management
  • Notifications inbox
  • Light & dark modes · AR & EN

◆ OUT OF SCOPE (v1.0)

  • Gym owner / admin app (separate project)
  • Real payment gateway integration (stub only)
  • Social features (friends, sharing)
  • Class booking / scheduling
  • Personal trainer marketplace
  • Wearable / fitness tracker integration
  • Web version
